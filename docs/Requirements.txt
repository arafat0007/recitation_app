# Final App Requirements Document

## Project Name

**Mother’s Islamic Audio App**
Modules: **Surah**, **Doa**, **Owaj**

---

# 1. Project Goal

Build a **very simple Android app for elderly users** that allows them to:

* listen to **Surah recitation**
* listen to **Doa recitation**
* read **Bengali meaning**
* optionally read **Arabic text**
* play **Owaj** videos from YouTube
* track **Watched/Not Watched** status for videos
* save **bookmarks/favorites**
* adjust **speed, theme, text size**, and core usability settings
* receive **dynamic content updates** from the app creator without requiring a new app release
* maintain **individual user progress** via a simple login

The app must prioritize:

* very simple interaction
* large buttons
* minimal confusion
* quick access to content
* Bengali UI
* easy maintenance by the creator

---

# 2. Locked Version 1 Decisions

These are fixed for Version 1 and should be treated as implementation constraints.

## 2.1 Platform

* Android only
* portrait mode only

## 2.2 UI Language

* App UI language: **Bengali only**
* English meaning: **not included in Version 1**
* Arabic text: **supported where content is available**

## 2.3 Surah and Doa Content

* Required:

  * Arabic recitation audio
  * Bengali meaning text
* Optional:

  * Arabic text display
  * Bengali explanation / tafsir / related notes as text
  * Bengali meaning audio, if content is available

## 2.4 Owaj Source & Progress

* **YouTube only** in Version 1
* **User-specific progress tracking**: "Watched" and "Not Watched" filtering.

## 2.5 Remote Content Strategy

* Remote source will manage:

  * titles
  * Bengali meaning
  * Arabic text
  * audio URLs
  * YouTube IDs
  * thumbnail URLs
  * sort order
  * publish status
* Creator updates content outside the mobile app

## 2.6 Offline Scope

* Settings and favorites must work offline
* Last fetched content should remain visible offline
* Auth session persistence should allow offline startup
* Offline media download is **not included** in Version 1

## 2.7 Implementation Priority

1. basic Android project foundation
2. **User Authentication (Login/Register)**
3. basic elderly-friendly UI design
4. home screen with the 3 main modules visible
5. **Owaj selection screen (Watched/Not Watched)**
6. **Paginated Owaj list screen**
7. Owaj detail with **Watched toggle** and playback
8. remote Owaj / YouTube list management
9. favorites and settings integration
10. Surah and Doa modules
11. Surah and Doa audio playback
12. remaining polish, testing, and release work

# 3. Primary Users

## 3.1 End User

**Elderly mother / elderly listener**

Characteristics:

* may struggle with precise touch gestures
* prefers large text and large tap targets
* needs very few steps to start playback
* prefers Bengali-first usage
* may not be comfortable with deep navigation

## 3.2 Admin / Creator

**App creator (you)**

Responsibilities:

* add, edit, delete Surah / Doa / Owaj items
* reorder items
* publish / unpublish items
* update remote lists without releasing a new APK

# 5. Product Vision

The app should feel like a **simple home screen with 3 large choices**.

A user should be able to:

1. open the app (auto-login if session exists)
2. tap **Surah**, **Doa**, or **Owaj**
3. for Owaj, choose **Watched** or **Not Watched**
4. tap an item (paginated navigation if needed)
5. press one large **Play** button
6. toggle **Watched** status if desired

Everything else should remain secondary.

# 6. Functional Requirements

## A. Authentication Requirements

### FR-A1. User Registration
* The app shall allow new users to register with **Name, Email, and Password**.
* User info shall be stored in a `users` collection in Firestore.

### FR-A2. User Login
* The app shall allow users to log in with **Email and Password**.
* The password field shall include a **visibility toggle** icon.
* The app shall provide clear **Bengali error messages** for "User not found" or "Incorrect password".

### FR-A3. Password Reset
* The app shall allow users to request a **password reset email** if they forget their password.

### FR-A4. Persistent Login
* The app shall remember the user's login session so they don't have to log in every time.

---

## B. Home Screen Requirements

### FR-B1. Home Layout
The home screen shall show:
* app title
* 3 large buttons/cards: Surah, Doa, Owaj
* 1 entry point to Favorites
* 1 entry point to Settings

---

## E. Owaj Module Requirements

### FR-E1. Owaj Selection Screen
* When entering the Owaj module, the user shall choose between:
  * **Not Watched (না দেখা ওয়াজ)**
  * **Watched (দেখা ওয়াজ)**

### FR-E2. Paginated Owaj List
* The app shall display Owaj items in a list of **20 items per page**.
* Large **"Next" (পরের)** and **"Previous" (আগের)** buttons shall be at the bottom.

### FR-E3. Owaj Detail and Progress
* The detail screen shall show a **"Watched?" (দেখা হয়েছে?)** toggle with **YES (হ্যাঁ)** and **NO (না)** buttons.
* Tapping YES/NO shall update the user's progress for that specific video.

---

## K. Data Requirements

### FR-K1. User Data
The system shall store:
* `users`: id, name, email, createdAt
* `user_video_status`: userId, owajId, isWatched, updatedAt
